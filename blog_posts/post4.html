My Project Specifications

In this blog I discussed about my project specification in details. It includes the user stories and user flow. I have also talked about the project's frontend, backend and database features.

User Stories

Type of users:

• System Administrator

• Registered Users

• Guest Users

System Administrator

1. As a System Administrator, I want to manage the product catalog so that I can add or remove items.

2. As a System Administrator, I want to manage users accounts so that I can update user information and add or delete users accounts.

3. As a System Administrator, I want to manage payments so that I can check if payments have been received or not.

4. As a System Administrator, I want to manage orders so that I have access to order status.

5. As a System Administrator, I want to manage the website so that I can update information and fix any issues.

User Flow:

Registered Users

1. As a Registered User, I want to browse listed products so that I can make my choice and buy them.

2. As a Registered User, I want to read details of each products so that I can make my choice to buy them or not.

3. As a Registered User, I want to add items to the shopping cart so that I can compare products, make adjustments and buy items.

4. As a Registered User, I want to go to the checkout page from the shopping cart page so that I can make payments.

5. As a Registered User, I want to make payments so that I can buy the products and it gets shipped to me.

User Flow:

Guest Users

1. As a Guest User, I want to be able to browse products so that I can buy them.

2. As a Guest User, I want to be able to register so that I can add items to cart and buy them later.

3. As a Guest User, I want to be able to make payments so that I can buy products.

User Flow:

Frontend Features

1. Sign Up form view: New users can create their account by providing their name, email address, and password into the sign-up form.

2. Login form view: Users can log into their accounts using their email and password on the login form.

3. General website view: The website will have a welcome page describing the product and separate pages for the products, contact, and shopping cart. User can login and logout by clicking the buttons.

4. Product items view: Each item can be opened in a new page containing all the details of the product like price, description, usage.

5. Shopping cart list view: The shopping cart can add or remove items from the list with each item in a separate row.

6. Payment form view: User can pay for the products they want buy using the payment form by providing their card details.

Backend Features

1. User authentication: There can be three types of users: guest, registered and administrator. Each of them will have different level of access to the website. Each registered customer will have their own account which they can manage.

2. Order management: Once an order is placed the admin can access the its status, shipping orders and customer information.

3. Product management: The administrator can add or remove products from the product catalog and update information.

4. Shopping cart management: The shopping cart will update as a customer adds or removes items and it should be emptied once the payment is made.

5. Administrator management: The administrator will have access to all parts of the website and can manage them.

Database Features

1. Product information: The database will hold and update detailed product descriptions, prices, specifications, id numbers, etc.

2. User’s personal information: Through databases, customer information such as contact details, names, spending patterns, and more are stored.

3. Transaction information: The database will manage and track every order throughout the sales in an organized manner.

4. Update inventory: The database keeps the inventory up to date after each transaction, such as what is in or out of stock, billing, shipping statuses, purchase orders, and more.

5. Shopping cart information: The database stores the product information of the shopping cart even if the user logs out of their account. It will still be there when the user logs back in.

6. Order history: The database also stores the previous orders of customers.
